## Контрольные вопросы

### 1. Какие типы данных поддерживает SQLite? Как в SQLite можно хранить логические значения и даты?

SQLite поддерживает пять основных типов (аффинитетов): `NULL`, `INTEGER`, `REAL`, `TEXT`, `BLOB`.  
Логические значения обычно хранятся как `INTEGER` (0 — false, 1 — true). Даты могут храниться как `TEXT` (формат ISO8601), `INTEGER` (Unix timestamp) или `REAL` (юлианские дни).

---

### 2. Для чего нужен класс SQLiteOpenHelper? Опишите назначение методов onCreate() и onUpgrade().

`SQLiteOpenHelper` — вспомогательный класс для управления созданием и обновлением базы данных.  
- `onCreate()` вызывается при первом создании базы данных; здесь выполняются запросы на создание таблиц.  
- `onUpgrade()` вызывается при увеличении версии БД; используется для изменения структуры (миграции) — например, для удаления старых таблиц и создания новых.

---

### 3. В чем разница между методами getWritableDatabase() и getReadableDatabase()? В каких ситуациях может возникнуть ошибка при вызове getWritableDatabase()?

- `getWritableDatabase()` возвращает объект `SQLiteDatabase` для чтения и записи.  
- `getReadableDatabase()` возвращает объект для чтения (если запись невозможна, но чтение доступно).  

Ошибка при вызове `getWritableDatabase()` может возникнуть, если диск заполнен, отсутствуют права на запись или база данных открыта в режиме только для чтения.

---

### 4. Что такое Cursor? Как правильно перемещаться по его элементам и почему важно закрывать его после использования?

`Cursor` — интерфейс для доступа к результатам запроса. Он позволяет перемещаться по строкам и читать значения столбцов.  
Для перемещения используются методы: `moveToFirst()`, `moveToNext()`, `moveToPosition()` и др. Обычно цикл выглядит так:  
```java
if (cursor.moveToFirst()) {
    do {
        // чтение данных
    } while (cursor.moveToNext());
}
